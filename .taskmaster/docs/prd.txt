# Order Management System - Feature Enhancement PRD

## Project Overview

This PRD covers two key UI/UX enhancement features for the order management system:
1. Kanban Board Feature - Visual drag-and-drop order management
2. Source Display Relocation - Improved source field UX

## Feature 1: Kanban Board Implementation

### Objective
Transform the traditional table-based order management view into a visual, drag-and-drop workflow system using existing order status system.

### Technical Requirements
- **Technology Stack**: Use existing @dnd-kit libraries, Radix UI components, Tailwind CSS
- **State Management**: Integrate with existing PocketBase real-time subscriptions
- **TypeScript**: Fully typed with custom interfaces for Kanban-specific data structures

### Components to Create
1. **KanbanCard.tsx** - Individual order card component with:
   - Order number (clickable to copy)
   - Customer name and phone (clickable to copy)
   - Order amount with currency formatting
   - Product count and total items
   - Creation date
   - Source marketplace with color coding
   - Quick action buttons (Details, Archive)
   - Communication options (Call, Viber, Telegram)

2. **KanbanColumn.tsx** - Status-based column component with:
   - Dynamic column creation based on available statuses
   - Color-coded headers matching status colors
   - Order count per column
   - Drag & drop drop zones

3. **KanbanBoard.tsx** - Main board component with:
   - Drag & drop logic using @dnd-kit sensors
   - Status update handling via existing handleStatusChange function
   - Real-time subscription integration
   - Marketplace API synchronization

4. **types.ts** - TypeScript interfaces for:
   - Kanban-specific data structures
   - Drag & drop event types
   - Column and card configurations

5. **utils.ts** - Utility functions for:
   - Drag & drop handlers
   - Status column generation
   - Order filtering and sorting

### Integration Requirements
- **Orders Dashboard**: Add toggle between table and Kanban views
- **Status Management**: Reuse existing status options and colors
- **Marketplace Sync**: Maintain existing marketplace status synchronization
- **Order Details Modal**: Same modal opens when clicking order cards
- **Archive Functionality**: Same archive behavior as table view
- **Internationalization**: Support existing i18n system

### Performance Considerations
- Efficient rendering with virtualization for large order sets
- Optimized drag operations with proper memoization
- Minimal re-renders through React optimization

### Accessibility Requirements
- Full keyboard navigation support via @dnd-kit
- Screen reader compatibility
- High contrast mode support
- Focus management during drag operations

## Feature 2: Source Display Relocation

### Objective
Improve UX by relocating source field from editable dropdown in order details modal to read-only badge in dialog header.

### Current Issues
- Source dropdown appears in Order Details Modal as editable field (lines 684-708)
- Allows unnecessary modification of source after order creation
- Creates UI clutter in modal body

### Technical Requirements

#### Remove Source Dropdown
- **Target File**: `order-details-modal.tsx`
- **Location**: Lines 684-708 in "Order Information" section
- **Action**: Delete entire source Select component section

#### Add Source Badge to Header
- **Location**: Dialog header section (around line 545)
- **Implementation**: 
  - Use existing `getSourceColor()` function for consistent styling
  - Find source name using: `sources.find(s => s.id === order.source)?.name`
  - Position near order title for immediate identification
  - Use existing Badge component pattern

#### Maintain Create Modal Unchanged
- **File**: `OrderCreate.tsx` (lines 172-192)
- **Action**: No changes required
- **Reason**: Source selection correctly placed during order creation

### Code Pattern Requirements
```tsx
// Use existing getSourceColor function
const sourceColor = getSourceColor(order.source)

// Find source name from sources array  
const sourceName = sources.find(s => s.id === order.source)?.name

// Badge pattern (similar to existing status badge)
<Badge 
  className="text-white text-xs"
  style={{
    backgroundColor: sourceColor,
    borderColor: sourceColor
  }}
>
  {sourceName}
</Badge>
```

### Benefits
1. **Cleaner Interface**: Removes unnecessary editable field from details modal
2. **Immediate Identification**: Source immediately visible in header
3. **Prevent Accidents**: No accidental source changes after creation
4. **Consistent Pattern**: Follows existing header badge pattern (status badge)
5. **Logical Separation**: Editable during creation, read-only during viewing/editing
