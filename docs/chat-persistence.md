# Chat Persistence Feature

This document describes the chat persistence feature implemented in the Order Management System, which allows chat conversations with the AI assistant to be saved and retrieved across sessions.

## Overview

The chat persistence system uses PocketBase as the storage backend to save and load chat messages. This implementation follows the Vercel AI SDK guidelines for chatbot message persistence.

## Features

- Start new chats
- Continue existing chats
- Chat history persisted across sessions
- Handles client disconnects gracefully
- Automatically creates chats when they don't exist
- Robust error handling

## PocketBase Integration

- **Native ID Generation**: PocketBase automatically generates unique IDs for chat records, eliminating the need for custom ID generation and validation
- **JSON Storage**: Chat messages are stored as JSON data in PocketBase, allowing for structured data storage and retrieval
- **Automatic Record Creation**: If a chat with a specific ID doesn't exist, a new chat is automatically created with a PocketBase-generated ID

## Why PocketBase vs. File-based Storage

We chose PocketBase over file-based storage for the following reasons:

**PocketBase Advantages:**
- **Consistent with existing system**: The application already uses PocketBase for other data storage
- **Scalability**: Better performance with a large number of chats
- **Query capabilities**: Allows filtering, searching, and indexing chat data
- **Security**: More secure than file system access in production environments
- **Concurrent access**: Handles multiple simultaneous read/write operations
- **Database features**: Transactions, backup, and data integrity
- **Future extensibility**: Easy to add user associations, metadata, and other features

**File-based Storage Limitations:**
- Limited scalability
- Security concerns with file system access
- Poor performance with many files
- Lack of query capabilities
- No built-in handling for concurrent access
- Challenging to backup and maintain

## Technical Implementation

### Database Schema

**Collection: `chats`**
- `id` (string): Primary key, auto-generated by PocketBase
- `messages` (json): Array of chat messages
- `created` (date): When the chat was created
- `updated` (date): When the chat was last updated

### Components

1. **Chat Storage** (`app/lib/chat-store.ts`):
   - `createChat()`: Creates a new chat and returns its PocketBase-generated ID
   - `chatExists(id)`: Checks if a chat exists in the database
   - `loadChat(id)`: Loads chat messages for a specific chat ID
   - `saveChat({ id, messages })`: Saves messages to a chat, creating a new one with a PocketBase-generated ID if needed

2. **API Route** (`app/api/chat/route.ts`):
   - Receives chat messages from the client
   - Processes them with the AI model
   - Saves the messages and AI responses to PocketBase
   - Handles streaming of AI responses
   - Returns the actual chat ID in headers (which might be a new ID if the original chat wasn't found)

3. **Chat Pages**:
   - `app/[locale]/chat/page.tsx`: Creates a new chat and redirects to the chat page
   - `app/[locale]/chat/[id]/page.tsx`: Loads an existing chat and displays its messages

4. **Chat Component** (`app/components/features/dashboard/AiChatBox.tsx`):
   - Displays chat messages
   - Handles user input
   - Maintains chat state
   - Uses the chat ID for persistence
   - Handles chat ID changes (when a new chat is created)

### Error Handling

The implementation includes robust error handling:
- If a chat doesn't exist when trying to save messages, a new chat is automatically created
- If saving a chat fails, the error is logged but doesn't affect the user experience
- If loading a chat fails, an empty array of messages is returned

## Setup

1. Create the PocketBase collection:
   ```bash
   npx ts-node scripts/setup-chat-collection.ts
   ```

2. Ensure PocketBase is running and accessible via the URL specified in your environment variables.

3. Test the chat persistence functionality:
   ```bash
   npx ts-node scripts/test-chat-store.ts
   ```

## Usage

1. Navigate to the chat feature using the ChatNavButton component or directly visit `/{locale}/chat`.
2. The system will create a new chat and redirect you to the chat page.
3. Your messages and the AI's responses will be saved automatically.
4. You can return to a specific chat by using its ID in the URL: `/{locale}/chat/{id}`.

## Error Handling

The implementation includes error handling for:
- Failed saves
- Failed loads
- Client disconnects
- Non-existent chats

If the client disconnects during a response, the system will continue to process the AI response and save it to the database, ensuring that when the client reconnects, they will see the complete response.

## Performance Considerations

- Chat messages are stored as a JSON string in PocketBase
- For very long conversations, consider implementing pagination
- The API endpoint uses `consumeStream()` to handle client disconnects

## Common Issues and Solutions

### 404 Not Found Error

If you see a 404 error when saving chat messages:
```
Error: The requested resource wasn't found.
```

This indicates the chat record doesn't exist in PocketBase. The system will automatically create a new chat with a different ID. The client will be informed of this ID change via the `X-Chat-ID` header.

### JSON Parsing Issues

If you encounter JSON parsing errors, ensure that the `messages` field is properly defined as a JSON field in PocketBase with a minimum length of 2 (for empty array "[]").
